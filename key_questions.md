# Ответы на ключевые вопросы

## Проблема  
Организация контекстно-зависимого автодополнения кода с пояснениями для студентов на основе педагогически адаптированного взаимодействия с нейросетевой моделью.

## Актуальность  
Современное обучение программированию всё чаще происходит в профессиональных интегрированных средах разработки (IDE), таких как Visual Studio Code. При этом исследования показывают, что своевременная помощь ИИ агентов при написании кода может значительно снизить когнитивную нагрузку, повысить мотивацию и ускорить усвоение синтаксических и концептуальных основ [1].

Однако существующие инструменты автодополнения ориентированы преимущественно на опытных разработчиков и генерируют код без пояснений, что делает их малополезными для студентов. Опросы показывают, что многие студенты не хотят использовать ИИ агентов при обучении программированию в связи с опасениями относительно академической добросовестности и последствий для будущего обучения [2]. 

В то же время, современные языковые модели обладают потенциалом не только генерировать код, но и объяснять его, если соответствующим образом сформулирован запрос (промпт). Это позволяет реализовать подход, при котором IDE передаёт текущий контекст в LSP-сервер, который через заранее подготовленный педагогический промпт направляет нейросетевую модель на генерацию не просто корректного, а образовательно полезного кода: с краткими комментариями, аналогиями и упрощёнными примерами [3].

Таким образом, разработка плагина, обеспечивающего автодополнение кода на основе учебного контекста, является актуальной задачей, способной повысить качество образования в области программирования.

## Объект исследования  
Инструмент контекстного автодополнения кода в средах разработки, применяемый в учебных проектах.

## Предмет исследования  
Наличие и качество пояснений в автодополнениях, генерируемых плагином на основе локального контекста кода в IDE.

## Цель дипломного исследования  
Разработать плагин для VS Code, обеспечивающий автодополнение кода с пояснениями для студентов за счёт интеграции LSP-сервера и нейросетевой модели с педагогически настроенным промптом.

## Цель на текущий семестр  
Спроектировать архитектуру LSP-сервера с использованием нейросетевой модели для обеспечения контекстно-зависимого автодополнения кода в учебных сценариях.

## Задачи  
1. Провести анализ существующих решений автодополнения кода в средах разработки.  
2. Определить состав локального контекста, доступного внутри VS Code и подходящего для передачи в нейросетевую модель.  
3. Разработать педагогически ориентированный промпт.  
4. Изучить структуру взаимодействия плагина и LSP-сервера с использованием нейросетевой модели.
