**LSP (Language Server Protocol)** — это протокол между редактором или IDE и языковым сервером, который расширяет работу с текстовым документом. **Языковой сервер** предназначен для предоставления интеллектуальных функций, специфичных для языка, и взаимодействия со средствами разработки по протоколу, который обеспечивает межпроцессное взаимодействие. Языковой сервер работает как отдельный процесс, а средства разработки взаимодействуют с сервером с помощью языкового протокола через JSON-RPC. Таким образом, протокол поддерживает ряд функций для удобного редактирования текста: автодополнение, переход к объявлению, поиск всех вхождений и др. Основная идея LSP — стандартизировать взаимодействие между языковым сервером и инструментом разработки (IDE). 

Задача в рамках дипломной работы - расширить протокол, добавив новые кастомные запросы между IDE и языковым сервером. Результат работы - плагин, реализующий автодополнение для языка python в среде разработки VS Code. Для реализации продукта необходимо создать:
1. **Language Server** — отдельная программа (процесс), которая работает в фоне и способна принимать и обрабатывать не только базовые, но и кастомные запросы.
2. **VS Code Extension** (Расширение) — это плагин для редактора VS Code, который работает внутри процесса VS Code и устанавливает связь с языковым сервером через LSP.
